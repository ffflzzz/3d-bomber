# UI/UX 开发指令：3D Bomper

> 本文档是给开发团队的前端 UI/UX 详细规范

---

## 一、视觉风格总则

### 1.1 风格定位

参考风格（按优先级）：
1. **3D Bomberman** — 经典炸弹人 3D 化的视觉感受
2. **Super Mario Odyssey** — 色彩鲜艳、角色可爱的 3D 风格
3. **Among Us** — 简洁的 3D 角色设计

**核心原则：**
- 3D 角色 + 2D UI 混合
- 色彩鲜艳但不刺眼
- 低多边形（low-poly）角色模型
- 清晰的视觉反馈（爆炸、受伤、拾取）

### 1.2 色彩方案

```
主色调：
  金色     #FFD700  — 英雄主题色（巴赫）
  深蓝     #1A237E  — UI 背景
  深灰     #212121  — 次要 UI 背景

辅助色：
  红色     #D32F2F  — 危险/敌人/爆炸
  绿色     #388E3C  — 安全/生命值
  蓝色     #1976D2  — 道具/技能
  紫色     #7B1FA2  — Boss/特殊

文字色：
  白色     #FFFFFF  — 主要文字
  浅灰     #BDBDBD  — 次要文字
  金色     #FFD700  — 标题/高亮
```

### 1.3 字体

- 中文：思源黑体（Noto Sans SC）
- 英文：Inter
- 游戏内标签：Press Start 2P（像素风）
- 标题：Playfair Display（衬线，巴洛克感）

## 三、UI 组件规范

### 3.1 面板

**样式：**
- 背景：深蓝渐变 (#1A237E → #0D47A1)
- 边框：金色 (#FFD700)，2px 实线
- 圆角：8px
- 阴影：0 4px 20px rgba(0,0,0,0.5)
- 内边距：16px

**尺寸：**
- 英雄选择面板：宽度 800px，高度 600px
- 设置面板：宽度 500px，高度 400px
- 任务日志：宽度 400px，高度 500px

### 3.2 按钮

**样式：**
- 背景：深蓝 (#1A237E)
- 边框：金色 (#FFD700)，1px
- 圆角：4px
- 内边距：8px 16px
- 文字：白色，14px，加粗

**状态：**
- 正常：背景 #1A237E，文字 #FFFFFF
- 悬停：背景 #283593，文字 #FFD700
- 点击：背景 #0D47A1，文字 #FFFFFF
- 禁用：背景 #424242，文字 #757575

### 3.3 图标

**尺寸：**
- 道具图标：32x32 px
- 技能图标：48x48 px
- 英雄头像：64x64 px
- 怪物头像：32x32 px

**风格：** 扁平化，带金色边框

### 3.4 血条/进度条

**样式：**
- 背景：深灰 (#424242)
- 前景：绿色 (#388E3C)
- 高度：8px
- 圆角：4px
- 边框：1px 深灰

**HP 条（角色上方）：**
- 宽度：48px（3D 世界中跟随角色）
- 高度：4px
- 红色 (#D32F2F) 当 HP < 30%

### 3.5 小地图

**样式：**
- 尺寸：120x120 px
- 背景：半透明黑色 (rgba(0,0,0,0.7))
- 边框：金色 1px
- 玩家标记：蓝色圆点
- 怪物标记：红色圆点
- 道具标记：黄色圆点
- 传送门标记：紫色圆点

### 4.2 英雄选择

**布局：** 横向卡片式

- 卡片下方：英雄名称、称号、emoji
- 点击卡片展开详细信息（属性面板、技能说明）
- 选中卡片高亮（金色边框）

**英雄卡片：**
```

│                  │
│    🎻 巴赫        │

│  DEF:■■■□□  SPD:■■■□□│
└──────────────────┘
```

### 4.3 暂停菜单

**触发：** 按 ESC 或点击暂停按钮
**布局：** 半透明遮罩 + 居中面板
- 继续游戏
- 设置
- 返回主菜单
- 退出游戏（二次确认）

### 4.4 游戏结束

**触发：** HP ≤ 0
**布局：** 全屏遮罩
- 大字标题："GAME OVER"
- 统计：击杀数、到达维度、用时
- 按钮：重新开始、返回主菜单

### 5.2 技能栏

**布局：** 底部居中，4 个技能槽
```
┌────┐ ┌────┐ ┌────┐ ┌────┐
│ Q  │ │ W  │ │ E  │ │ R  │
│图标│ │图标│ │图标│ │图标│
```

**样式：**
- 未就绪：半透明
- 就绪：全彩 + 微光效果
- 冷却中：灰色遮罩 + 数字倒计时

### 5.3 通知系统

**类型：**
1. 道具拾取通知（右上角，2 秒消失）
2. 升级通知（屏幕中央，3 秒消失）
3. 任务完成通知（屏幕中央，3 秒消失）
4. 维度切换通知（全屏淡入淡出，2 秒）

**样式：**
- 背景：半透明黑色
- 文字：白色/金色
- 动画：从上方滑入，停留后滑出

## 六、移动端适配

### 6.1 触屏控制

**虚拟摇杆：**
- 位置：屏幕左下角
- 尺寸：直径 100px
- 响应区域：摇杆周围 50px 范围
- 拖动范围：半径 40px

**动作按钮：**
- 位置：屏幕右下角
- 尺寸：直径 60px
- 按钮：放置炸弹、释放技能
- 长按：技能充能

### 6.2 响应式布局

**断点：**
- 桌面 (>768px)：完整 UI
- 平板 (481-768px)：缩小 HUD，隐藏部分面板
- 手机 (<480px)：仅保留核心 HUD，技能栏改为底部滑动

**触摸目标：**
- 最小触摸区域：44x44 px
- 按钮间距：至少 8px

### 6.3 性能分级

**设备检测：** (`platform/deviceProfile.ts`)
- High-end：全粒子效果，高纹理分辨率
- Mid-range：中等粒子效果，中等纹理
- Low-end：关闭粒子效果，低纹理分辨率，简化阴影

## 七、动画与过渡

### 7.1 页面切换

- 主菜单 → 英雄选择：淡入淡出 (300ms)
- 英雄选择 → 游戏：缩放淡入 (500ms)
- 游戏 → 暂停：半透明遮罩 (200ms)

### 7.2 游戏内动画

- 爆炸：径向扩散 + 粒子喷射 (500ms)
- 道具拾取：向上浮动 + 缩小 (300ms)
- 受伤：屏幕闪红 (100ms)
- 传送门激活：漩涡旋转 (2s)

### 7.3 英雄出场动画

- 从屏幕上方降落
- 落地时产生冲击波
- 显示英雄名称和称号
